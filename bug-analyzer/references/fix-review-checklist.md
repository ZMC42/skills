# Fix Review Checklist

按需在以下场景读取本文件：

- 用户要求评估某个 fix 是否彻底
- 你怀疑修复只是掩盖症状
- 你需要输出回归风险和改进建议

## 修复有效性

逐项检查：

- 修复是否直接切断了原始触发链路
- 修复是否依赖“碰巧不会再触发”而不是显式约束状态
- 修复是否只在表现层加了 try/catch、默认值、吞错或 early return
- 修复是否补上了缺失的输入校验、状态迁移保护或资源释放

## 边界覆盖

重点追问：

- 空值、空集合、零值、超长值是否覆盖
- 单线程正常，多线程或重试下是否仍成立
- 首次执行与重复执行语义是否一致
- 超时、取消、回滚、失败重试路径是否覆盖
- 上下游返回脏数据或旧配置时是否仍安全

## 回归风险

常见新增风险：

- 新判空改变旧有业务语义
- 新锁或同步扩大临界区，导致吞吐下降或死锁风险
- 新重试导致重复写入、重复消费或幂等性破坏
- 新缓存/回退逻辑掩盖真实错误
- 修复只覆盖当前模块，未同步上下游契约

## 输出建议

如果修复充分，明确写：

- 修复覆盖了哪个触发点
- 为什么它也覆盖了上游异常状态
- 哪些边界已被处理

如果修复不充分，明确写：

- 当前 fix 解决了什么症状
- 根因还有哪部分未被消除
- 更合理的修复点应落在哪个模块、条件或状态节点
